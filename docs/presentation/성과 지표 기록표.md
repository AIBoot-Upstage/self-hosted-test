# 성과 지표 기록표

목표와 실측 결과를 같은 칸에 섞지 않는다. 표본이 부족하면 숫자 대신 `baseline pending`을
유지하고, 실측값에는 표본 수와 측정 기간을 함께 기록한다.

## 1. 측정 정보

| 항목 | 값 |
| --- | --- |
| 측정 기간 | `[YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm KST]` |
| 서비스 image tag / commit SHA | `[값 입력]` |
| 대상 저장소와 PR 수 | `[값 입력]` |
| Small / Medium / Large 표본 수 | `[N / N / N]` |
| Solar3 model | `solar-pro3` |
| 정책 index 문서 / chunk 수 | `[N / N]` |

## 2. 운영 신뢰성

| 지표 | 계산식 | 목표 | 실측 |
| --- | --- | ---: | ---: |
| 유효 webhook 수락률 | 유효 요청 중 2xx / 전체 유효 요청 | ≥ 99.5% | baseline pending |
| 자동 리뷰 완료율 | 완료된 eligible head SHA / 전체 eligible head SHA | ≥ 98.0% | baseline pending |
| 10분 내 일시 오류 복구율 | 재시도 성공 / timeout·rate-limit 실패 | ≥ 95.0% | baseline pending |
| 중복 리뷰율 | 중복 repo·PR·head·mode / 전체 실행 | ≤ 0.5% | baseline pending |
| 결과 schema 유효율 | parse·필수 필드 통과 / 전체 LLM 응답 | ≥ 99.0% | baseline pending |

Draft, 사용자 취소, 새 commit으로 대체된 head는 완료율 분모에서 제외하고 `canceled`로 별도
기록한다. 외부 API 장애는 실패 유형으로 기록하며 임의로 분모에서 제거하지 않는다.

## 3. Latency

`ci_wait_ms = check_suite.completed - PR head 생성`

`agent_latency_ms = AI Check 완료 - check_suite.completed webhook 수신`

| Diff 구간 | 기준 | p50 목표 | p95 목표 | p50 실측 | p95 실측 | N |
| --- | --- | ---: | ---: | ---: | ---: | ---: |
| Small | ≤ 200 lines, ≤ 5 files | ≤ 30초 | ≤ 75초 | pending | pending | 0 |
| Medium | ≤ 800 lines, ≤ 20 files | ≤ 60초 | ≤ 150초 | pending | pending | 0 |
| Large | ≤ 2,000 lines, ≤ 40 files | ≤ 120초 | ≤ 300초 | pending | pending | 0 |

두 기준 중 큰 구간을 사용한다. 초과 PR은 `oversized`로 분리하고 포함 파일 수와 truncation을
기록한다.

## 4. 라우팅과 RAG

| 지표 | 목표 | 실측 | 표본 |
| --- | ---: | ---: | ---: |
| Routing macro F1 | ≥ 0.95 | pending | `[N]` |
| Retrieval Recall@3 | ≥ 0.85 | pending | `[N]` |
| Retrieval MRR@3 | ≥ 0.75 | pending | `[N]` |
| Citation validity | ≥ 0.98 | pending | `[N]` |
| Unrelated retrieval rate | ≤ 0.05 | pending | `[N]` |

- Recall@3: 정답 정책이 검색 결과 상위 3개에 포함된 query 비율
- MRR@3: 첫 정답 정책 순위의 reciprocal rank 평균
- Citation validity: finding의 정책 출처가 실제 retrieved set에 존재하는 비율

Finding 일치는 `file + category + 핵심 issue label` 기준으로 두 명이 독립 판정하고, 불일치는
합의한다. 문장 유사도만으로 정답 처리하지 않는다.
